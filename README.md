# max-qt

Фикс пустого окна мессенджера **MAX** на MacBook Pro 11,4 и 11,5 (Mid 2015, 15", Iris Pro 5200 / Radeon R9 M370X).

## Проблема

После установки и запуска `Max.app` окно открывается, но содержимое не отрисовывается — пустое (белое / прозрачное) окно без интерфейса. MAX построен на Qt; по умолчанию Qt Quick Scene Graph использует Metal-бэкенд, который на встроенных GPU MacBookPro 11,x ведёт себя некорректно.

## Что делает скрипт

[`max-qt-repire.sh`](./max-qt-repire.sh) выполняет три шага:

1. **Прописывает `QSG_RHI_BACKEND=opengl`** в `Info.plist` приложения (секция `LSEnvironment`) через `PlistBuddy` — принудительно переключает Qt на OpenGL-бэкенд.
2. **Снимает карантин** (`xattr -dr com.apple.quarantine`) и **переподписывает** приложение ad-hoc подписью (`codesign --force --deep --sign -`) — правка `Info.plist` ломает исходную подпись, без переподписи Gatekeeper не даст запустить.
3. **Запускает** `Max.app`.

## Использование

```sh
chmod +x max-qt-repire.sh
./max-qt-repire.sh
```

По умолчанию скрипт работает с `~/Applications/Max.app`. Если приложение установлено в системную папку, отредактируйте первую строку:

```sh
APP=/Applications/Max.app
```

Во время выполнения может появиться запрос Связки ключей на доступ к ключу подписи — это нормально, разрешите.

## Требования

- macOS на MacBook Pro 11,4 / 11,5 (или другом устройстве, где MAX открывается пустым окном из-за Metal-бэкенда Qt).
- Установленный `Max.app`.
- Утилиты `PlistBuddy`, `xattr`, `codesign` — все идут с macOS, дополнительно ставить ничего не нужно.

## Откат

Достаточно переустановить `Max.app` — установщик перезапишет `Info.plist` и восстановит оригинальную подпись.
